Currently there is a base model, in which the values for the seven adjusted parameters are

\begin{align} \bullet \quad \gamma = 2 \text{ (CRRA elasticity of substitution)} \\ \bullet \quad \sigma_z = 0.0521 \text{ (SD of income process white noise)} \\ \bullet \quad \alpha = 0.14 \text{ (1 - share of income for consumption)} \\ \bullet \quad \theta = 0.25 \text{ (Down payment parameter)} \\ \bullet \quad r_{rent} = .0755 \text{ (Rental rate (?))} \\ \bullet \quad a_0 = 0 \text{ (Initial level of assets)} \\ \bullet \quad D_0 = 0 \text{ (Initial level of durables)} \\ \end{align}

For $\gamma$, $\alpha$, $\theta$ and $r_{rent}$ there is a "low" model that sets the parameter value one percent lower than its base value, and a "high" one that sets it one percent higher. $\sigma_z$ includes those as well as one setting it to 0.

The settings for the assets and durables are a bit more complicated, as will be seen later.

from itertools import chain

# There are 7 x 7 models overall, equally spaced grid
initM = ['a%.1fD%.1f' % (x, y) for x in np.linspace(0, 0.6, 7) 
                                for y in np.linspace(0, 1.2, 7)]

# Work getting to the table. Notice the model identifiers are split
# to retrieve separate parameters
base = lifecycle_iterate({'elasticity': 2}, baseFile)
base.appendModels('fthb_dist', model=initM)
base.appended['assets'] = base.appended['model'].str[:4]
base.appended['durables'] = base.appended['model'].str[4:]
base.appended.set_index(['id', 'assets', 'durables'], inplace=True)
out = base.appended.unstack('durables')['ageBought'].groupby(level=1).mean()

# Work getting to the heatmap
fig, ax = plt.subplots()
ax.pcolor(out, cmap=plt.cm.OrRd)

ax.set_xticks(np.arange(out.shape[0])+0.5, minor=False)
ax.set_yticks(np.arange(out.shape[0])+0.5, minor=False)

ax.set_xticklabels(np.linspace(0, 1.2, 7), minor=False)
ax.set_yticklabels(np.linspace(0, 0.6, 7), minor=False)
out
